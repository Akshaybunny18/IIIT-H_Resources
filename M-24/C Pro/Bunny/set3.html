Problem 1: Grade Computing (60 points)

In a school management system, there is a need to manage student grades based on operations provided in an input file. The system should keep track of each student's personal information and their scores in various subjects, calculate the total grade percentage, and print each student's grade summary in alphabetical order

The input file, named students.txt, contains a list of operations to be applied to items in the student portal. Each operation will tell you the marks of the student in a specific subject

Each line in the file follows one of the formats below:

  • Student Initialization: Student_Name Age Initialises the name and the age for a student.
  • Add Marks (Out of 100): Student_Name Subject Marks Update the student's marks for a specific subject. If there are multiple operations of adding marks to the same subject for a student, consider the highest one

Your task is to:

  1. Read and process each operation from the students.txt file.
  2. Maintain a record of each Student's marksheet
  3. In the end, calculate and return each student's total grade percentage (rounded to two decimal places).
  4. This should be returned in the alphabetical order of the name
  5. Note: Grade % will vary according to the number of subjects he has attempted

Constraints:

  • Each line in students.txt represents a valid operation
  • Each student has a unique name, and age is a positive integer.
  • Number of Operations <= 1000
  • 0 <= marks <= 100

Input Format:

The file students.txt contains lines formatted as follows: 

Student_Name Age
Student_Name Subject Marks

Output Format:

Output the item's details in the following format: 

Student_Name Grade_Percentage(Rounded to 2 decimals)

Example File:

Suppose students.txt contains the following:

John Doe 18
Jane 20
John Math 85
Jane Math 92
John English 78
Jane English 88
John Math 90  
Jane Science 95

Example Output:

Jane Smith 91.67
John Doe 84.00

Notes:

  • The program reads and parses each line of data, maintaining the current record of each student's subjects and marks.
  • For each subject, only the highest mark is kept if multiple entries exist.
  • The total grade percentage for each student is calculated based on the average of their highest scores in each subject, rounded to two decimal places.

Problem 2: Avoid K Palindrome (60 points)

You are given a string S of length N consisting only of lowercase English letters. Find the number of strings obtained by permuting the characters of S (including the string S itself) that do not contain a palindrome of length K as a substring
Here, a string T of length N is said to "contain a palindrome of length K as a substring" if and only if there exists a non-negative integer i not greater than (N−K) such that Ti+j = Ti+K+1-j for every integer j with 1 ≤ j ≤ K.
Here, TK denotes the k-th character of the string T.

Input Format:

  • First line of input contains 2 integers N and K, the number of letters in the string and the length of palindrome to be avoided.
  • The string S

Output Format:

  • Print 1 line containing an integer: the answer to the question

    • Constraints:

    • 1 ≤ N ≤ 10
    • 1 ≤ K ≤ N

    Example

    Input

    3 2
aab

    Output

    1

    Input

    10 5
abcwxyzyxw

    Output

    440640

    Problem 3: Library Inventory System (60 points)

    In a library management system, you need to maintain an inventory of books and handle various operations such as updating book quantities, initializing book prices, and calculating the total value of the books in the inventory.

    The system reads commands from a file, processes each operation, and updates the inventory accordingly.

    The input file, named library_inventory.txt, contains a list of operations to be applied to the books in the inventory. Each operation will either initialize a book's price, add or remove quantities of a book.

    Operations in the Input File

    Each line in the file follows one of the formats below:

    • Price Initialization: BookTitle Price – Sets the price for a book. If the book already exists, it updates the price.
    • Add/Remove Quantity: BookTitle Add Quantity or BookTitle Remove Quantity – Adds or removes a specified quantity of the book from the inventory. The quantity should not go below zero.

    Your Task

    Write a program that:

    1. Reads and processes each operation from the library_inventory.txt file.
    2. Maintains a record of each book's price, quantity, and total value in the library inventory.
    3. At the end of processing all commands, outputs the details for each book in the inventory, including:
      • Title
      • Quantity
      • Total Value (calculated as Price * Quantity and rounded to two decimal places)
    4. If a book's quantity becomes zero, its total value should be zero.

    Constraints:

    • Each line in library_inventory.txt represents a valid operation.
    • The number of operations ≤ 100.
    • The quantity for each Add or Remove operation is a positive integer.
    • The price for each book is between 0 < Book_Price <= 1000.00.
    • Each book title is unique (case-sensitive).

    Input Format

    The file library_inventory.txt contains lines formatted as follows:

    BookTitle Price
BookTitle Add Quantity
BookTitle Remove Quantity

    Output Format

    Output the details of all books at the end, displaying their title, quantity, and total value, in the following format:

    BookTitle Quantity TotalValue

    Example Input

    Suppose library_inventory.txt contains the following:

    Book1 15.0
Book2 20.0
Book1 Add 5
Book2 Add 3
Book1 Remove 2

    Example Output

    Book1 3 45.00
Book2 3 60.00

    Notes

    • The program reads and parses each line of the input file, maintaining the current price and quantity for each book.
    • The quantity should not go below zero. If the "Remove Quantity" operation would make the quantity negative, the quantity should remain zero.
    • The total value of a book is calculated as Price * Quantity and should be printed rounded to two decimal places.
